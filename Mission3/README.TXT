Pour tester les méthodes intermédiaires numDiv() et sumDiv()
(retournant respectivement le nombre de diviseurs et leur somme),
nous avons effectué des tests sur de petites valeurs du paramètre n avec des cas de figures variés :
l'entier un qui n'a qu'un diviseur, 2 qui est premier, 4 qui est carré et 6 qui est le plus petit nombre
à avoir deux facteurs premiers.
Nous sommes donc assez confiants sur la correction de ces deux méthodes.

Pour les méthodes parfait() et sublime(), le nombre d'entiers pour lesquels cette méthode renvoie true
de taille raisonnable est assez limité donc nous avons testé sur ces valeurs, ainsi que sur quelques
autres valeurs arbitraires.
Les résultats ont également été concluants.
